## Working Style
- Before implementation, restate the requested behavior as one-line rules and confirm only when behavior is sensitive.
- Then implement directly without long planning chatter.
- Run a compile check at the end and report only key results.
- When adjusting unit or building scale, first reference docs/starcraft-size-reference.md and preserve original StarCraft proportions where practical.
- When adjusting combat stats, first reference docs/starcraft-combat-reference.md and use BWAPI-backed values where practical.
- When adding a new unit, check docs/starcraft-size-reference.md and docs/starcraft-combat-reference.md first; if the unit is missing there, follow the external reference links in those docs before implementing the unit.
- If a unit or building is missing from docs/starcraft-size-reference.md, check the Liquipedia source linked there before implementing size-related changes.
