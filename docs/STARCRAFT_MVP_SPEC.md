# StarCraft-Style RTS MVP 명세

이 문서는 현재 코드베이스를 기준으로, "게임으로 성립하는 최소 기능(MVP)"을 정의합니다.

## 1. 목표

- 소규모 전투가 가능한 RTS 루프 완성
- 입력, 이동, 전투, 승패 판정까지 한 사이클 구현

## 2. MVP 범위

### 포함

- 유닛 선택/명령(이동, 공격이동, 정지)
- 경로 탐색과 기본 충돌 회피
- 원거리/근거리 유닛 전투
- HP/사망 처리
- 간단한 승리/패배 조건

### 제외(차기 단계)

- 건물/생산
- 자원 채취
- 테크 트리
- 멀티플레이

## 3. 기능 요구사항

## 3.1 입력

- 박스 선택, 단일 선택
- 우클릭 이동
- 우클릭 적 대상 지정 공격
- `A` + 클릭 공격이동
- `S` 정지

완료 기준:
- 선택된 유닛만 명령 수행
- 명령 전환 시 이전 상태(타겟/이동)가 일관되게 갱신

## 3.2 이동/경로 탐색

- Grid 기반 A* 경로 탐색
- 단위 시간당 속도 적용
- 단순 군집 상황에서 겹침 완화

완료 기준:
- 장애물/유닛 밀집 구간에서 장시간 정지 현상 최소화
- 목적지 근접 시 자연스럽게 정지

## 3.3 전투

- 유닛별 공격력/사거리/공속 분리
- 타겟 사망 처리 및 재탐색
- 공격/피격 시각 효과

완료 기준:
- 공속이 프레임 독립적으로 안정 동작
- 사망 직후 타겟 참조 오류 없음

## 3.4 게임 루프

- 고정 간격 업데이트 (현재 33ms)
- 렌더링 시 y축 정렬로 가림 순서 유지

완료 기준:
- 평균 프레임 유지
- 유닛 수 증가 시도에도 치명적 버벅임이 없음

## 3.5 승패

- 한 팀 유닛 전멸 시 결과 표시
- 재시작 또는 종료 선택 제공

완료 기준:
- 전멸 순간 중복 판정 없음
- UI가 1회만 생성되고 입력 가능

## 4. 데이터 모델 요구사항

- `Unit` 공통 스탯: `hp`, `damage`, `range`, `speed`, `attackDelay`, `team`
- 상태: `commandState`, `target`, `manualOrder`, `isMoving`
- 맵: `TerrainGrid` (cell size, walkable)

## 5. 품질 기준

- Null 타겟 참조/동시 제거 예외 없음
- 유닛 50~100개 규모에서 기본 플레이 가능
- 주요 입력 반응 지연이 체감되지 않을 것

## 6. 테스트 체크리스트

- `A` 명령 후 빈 바닥 클릭 시 Attack-Move 정상
- 적 클릭 공격 후 타겟 사망 시 재탐색 정상
- `S` 입력 시 즉시 정지
- 드래그 선택 시 아군 우선 선택 규칙 확인
- 근접/원거리 각각의 사거리 판정 확인

## 7. 현재 코드 매핑

- 입력: `src/starcraft/engine/InputHandler.java`
- 루프/렌더: `src/starcraft/core/GamePanel.java`
- 유닛 상태/공격: `src/starcraft/objects/Unit.java`
- 길찾기: `src/starcraft/engine/PathFinder.java`
- 유닛 타입: `src/starcraft/objects/units/*`
